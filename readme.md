# TeamFinder

**TeamFinder** — это веб-платформа для поиска команды и участников в проекты. Приложение позволяет пользователям создавать карточки проектов, искать единомышленников, добавлять интересные работы в избранное, а также управлять своим профилем и контактами.

## Используемые технологии

*   **Backend:** Python 3.10+, Django 5.x
*   **Database:** PostgreSQL
*   **Frontend:** HTML5, CSS3, Vanilla JavaScript (без использования тяжелых фреймворков)
*   **Image Processing:** Pillow (PIL) для программной генерации аватаров пользователей
*   **Containerization:** Docker, Docker Compose (для развертывания базы данных и приложения)
*   **Configuration:** `python-dotenv` для управления переменными окружения

## Локальный запуск проекта

Следуйте этой инструкции, чтобы запустить проект на вашем локальном компьютере.

### 1. Подготовка окружения

1.  **Клонируйте репозиторий** (если еще не сделали этого):
    ```bash
    git clone <url-вашего-репозитория>
    cd team-finder-ad
    ```

2.  **Настройка переменных окружения (.env):**
    
    В проекте используется файл `.env` для хранения конфиденциальных настроек. Скопируйте пример и заполните его:
    ```bash
    cp .env_example .env
    ```

    Откройте файл `.env` и проверьте значения. Пример содержимого:
    ```ini
    DJANGO_SECRET_KEY=django-insecure-change-me-to-random-string

    DEBUG=True

    POSTGRES_DB=teamfinder
    POSTGRES_USER=postgres
    POSTGRES_PASSWORD=postgres
    POSTGRES_HOST=db
    POSTGRES_PORT=5432

    TASK_VERSION=1

    ALLOWED_HOSTS=localhost,127.0.0.1,web
    ```

   
### 2. Запуск инфраструктуры и приложения

Проект полностью контейнеризирован. Для запуска требуется только установленный Docker.

1.  **Запустите все сервисы (БД и Web-приложение):**
    ```bash
    docker-compose up -d --build
    ```
    *   `-d`: запуск в фоновом режиме.
    *   `--build`: принудительная пересборка образа (полезно при первом запуске или изменении зависимостей).

2.  **Проверьте статус контейнеров:**
    ```bash
    docker-compose ps
    ```
    Вы должны увидеть два запущенных контейнера: `db` (PostgreSQL) и `web` (Django).

### 3. Миграции и создание данных

Так как приложение работает внутри контейнера, команды `manage.py` нужно выполнять через `docker-compose run`.

1.  **Примените миграции базы данных:**
    ```bash
    docker-compose run web python manage.py migrate
    ```

2.  **Создайте суперпользователя (администратора):**
    ```bash
    docker-compose run web python manage.py createsuperuser
    ```
    Следуйте инструкциям в терминале. Введите email, имя, фамилию, телефон и пароль.

   
### 4. Доступ к приложению

Откройте браузер и перейдите по адресу: **[http://127.0.0.1:8000](http://127.0.0.1:8000)**

Если вы видите главную страницу приложения — поздравляем, проект успешно запущен!

## Структура проекта

*   `team_finder/` — конфигурация проекта (`settings.py`, `urls.py`, `wsgi.py`).
*   `users/` — приложение пользователей:
    *   Модели (`User`), формы регистрации и редактирования профиля.
    *   Логика генерации аватаров (`utils.py`).
*   `projects/` — приложение проектов: модели, представления для создания и просмотра проектов.
*   `templates_var1/` — HTML-шаблоны (согласно варианту задания).
*   `static/` — статические файлы (CSS стили, JavaScript скрипты, изображения).
*   `media/` — директория для загружаемых файлов (аватары пользователей).
*   `docker-compose.yml` — конфигурация для запуска сервисов (БД и Web).
